================================================================================
  TYPESCRIPT CHO DART DEV - IMPORT/EXPORT VÀ ĐỌC TYPE ERRORS
================================================================================

THỜI LƯỢNG ƯỚC TÍNH:
  - 20-30 phút

📌 MỤC TIÊU:
  - Hiểu import/export để lần theo code Angular nhanh
  - Đọc file structure mà không bị rối
  - Nhận ra một số lỗi TypeScript phổ biến khi sửa code

  ⚠️ Trong lộ trình 2 tuần:
  - Bài này là CORE
  - Trong code Angular mới, standalone quan trọng hơn việc học sâu về NgModule

================================================================================
  1. EXPORT VÀ IMPORT
================================================================================

  // user.model.ts
  export interface User {
    id: number;
    name: string;
  }

  export class UserService {
    getUsers(): User[] {
      return [];
    }
  }

  // file khác
  import { User, UserService } from './user.model';

  Bạn cần đọc như sau:
  - export = file này công khai type/class/function này
  - import = file kia đang dùng lại nó

================================================================================
  2. NAMED EXPORT LÀ MẶC ĐỊNH NÊN DÙNG
================================================================================

  // logger.ts
  export class Logger {
    log(message: string): void {
      console.log(message);
    }
  }

  // file khác
  import { Logger } from './logger';

  Thực tế Angular:
  - thường dùng named export
  - giảm nhầm lẫn khi rename/symbol search
  - dễ lần theo component/service/model hơn

================================================================================
  3. BARREL EXPORTS (INDEX FILES)
================================================================================

  // models/index.ts
  export { User } from './user.model';
  export { Product } from './product.model';

  // usage
  import { User, Product } from './models';

  Bạn sẽ gặp trong codebase lớn:
  - shared models
  - shared UI exports
  - feature public API

================================================================================
  4. ĐỌC CẤU TRÚC FILE ANGULAR HIỆN ĐẠI
================================================================================

  src/app/
  ├── core/
  ├── shared/
  ├── features/
  │   └── products/
  │       ├── product-list.component.ts
  │       ├── product-form.component.ts
  │       ├── product.service.ts
  │       └── product.model.ts
  ├── app.config.ts
  └── app.routes.ts

  Cách lần code:
  - route -> component
  - component -> service
  - service -> model / HttpClient
  - form / template -> event -> state update

  ⚠️ NgModule có thể vẫn xuất hiện trong code cũ, nhưng không phải điểm vào
  codebase nhanh cho lộ trình này.

================================================================================
  5. 4 LỖI TYPE THƯỜNG GẶP KHI SỬA CODE
================================================================================

  Lỗi 1:
  Cannot find name 'X'
  -> thiếu import, gõ sai tên, hoặc file chưa export symbol này

  Lỗi 2:
  Module has no exported member 'X'
  -> import sai symbol, hoặc barrel file chưa re-export

  Lỗi 3:
  Object is possibly 'undefined'
  -> cần null check, default value, hoặc đổi flow control

  Lỗi 4:
  Type 'A' is not assignable to type 'B'
  -> payload/response/state shape đang sai contract

================================================================================
  ✅ SAU BÀI NÀY BẠN CẦN BIẾT:
  - Lần theo import/export để đọc feature nhanh
  - Đọc barrel file mà không bị rối
  - Không cần học NgModule trước khi làm app Angular mới
  - Xử lý được những type errors cơ bản nhất
