================================================================================
  APPENDIX - DECORATORS ĐỂ ĐỌC ANGULAR CODE
================================================================================

THỜI LƯỢNG ƯỚC TÍNH:
  - 10-15 phút

📌 MỤC TIÊU:
  - Chỉ đủ mental model để đọc @Component, @Injectable, @Input, @Output
  - Biết decorators trong Angular dùng để gắn metadata vào class/member
  - Không sa vào viết custom decorators quá sớm

  ⚠️ Trong lộ trình 2 tuần:
  - Bài này là OPTIONAL
  - Bạn không cần tự viết decorator để làm feature Angular thông thường

================================================================================
  1. DECORATOR LÀ GÌ?
================================================================================

  // Angular sẽ gặp rất nhiều:
  @Component({...})
  @Injectable({...})
  @Input()
  @Output()

  Cách hiểu để đọc code:
  - @Component: đây là component và cấu hình template/styles/imports
  - @Injectable: đây là service có thể inject
  - @Input/@Output: contract giao tiếp parent <-> child

================================================================================
  2. SO SÁNH NHANH VỚI FLUTTER
================================================================================

  Flutter:
  - annotation thường là metadata cho analyzer/codegen
  - ví dụ: @override, @deprecated, @riverpod, @freezed

  Angular:
  - decorator là API framework để đăng ký metadata
  - bạn đọc decorator để biết class này đóng vai trò gì trong app

================================================================================
  3. NHỮNG DECORATOR CẦN ĐỌC TRONG 2 TUẦN ĐẦU
================================================================================

  @Component({
    selector: 'app-header',
    standalone: true,
    templateUrl: './header.component.html',
    styleUrls: ['./header.component.css'],
  })
  export class HeaderComponent {}

  @Injectable({ providedIn: 'root' })
  export class AuthService {}

  export class ProductCardComponent {
    @Input() title: string = '';
    @Output() selected = new EventEmitter<void>();
  }

  Bạn không cần nhớ cơ chế runtime bên dưới.
  Bạn chỉ cần đọc ra:
  - component nào render UI
  - service nào có thể inject
  - component nhận/emit dữ liệu gì

================================================================================
  4. NOTE CHO ANGULAR MỚI
================================================================================

  Trong Angular mới, bạn có thể thấy thêm function APIs như:
  - input()
  - output()
  - inject()

  Nghĩa là:
  - không phải mọi thứ đều phải học qua custom decorator examples
  - ưu tiên hiểu public API framework và vai trò của từng file

================================================================================
  ✅ SAU BÀI NÀY:
  - Đọc được decorators phổ biến trong Angular
  - Không còn xem decorators là chủ đề bắt buộc phải học sâu
  - Tập trung vào component, service, router, forms, HTTP, signals trước
