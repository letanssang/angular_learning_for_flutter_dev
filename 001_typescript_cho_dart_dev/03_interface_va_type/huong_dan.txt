================================================================================
  TYPESCRIPT CHO DART DEV - INTERFACE, TYPE VÀ GENERICS CƠ BẢN
================================================================================

THỜI LƯỢNG ƯỚC TÍNH:
  - 30-40 phút

📌 MỤC TIÊU:
  - Hiểu interface và type để đọc model/service/component Angular
  - Biết 3 utility thực dụng nhất: Partial, Pick, Omit
  - Nắm generics cơ bản để đọc HttpClient và reusable helpers

  ⚠️ Trong lộ trình 2 tuần:
  - Bài này là CORE
  - Chỉ cần đủ để đọc và sửa code Angular thực tế
  - Utility types nâng cao, conditional types, mapped types để sau

================================================================================
  1. INTERFACE TRONG TYPESCRIPT
================================================================================

  // ──── DART ────
  abstract class Animal {
    String name;
    void speak();
  }

  // ──── TYPESCRIPT ────
  interface Animal {
    name: string;
    speak(): void;
  }

  ⚠️ KHÁC BIỆT QUAN TRỌNG:
  - TS interface chỉ tồn tại ở compile-time, runtime sẽ bị xóa
  - Interface dùng để mô tả "shape" của object
  - Angular dùng nó rất nhiều cho models, API contracts, DTOs

  class Dog implements Animal {
    name: string;

    constructor(name: string) {
      this.name = name;
    }

    speak(): void {
      console.log(`${this.name} says Woof!`);
    }
  }

================================================================================
  2. TYPE ALIAS
================================================================================

  type Point = {
    x: number;
    y: number;
  };

  type ID = string | number;
  type Callback = (data: string) => void;

  Khi nào dùng gì?
  - interface: model object, API response, public contract
  - type: union, function type, alias nhanh, utility type

================================================================================
  3. UNION, INTERSECTION, OPTIONAL, READONLY
================================================================================

  type Status = 'idle' | 'loading' | 'error';

  type HasName = { name: string };
  type HasAge = { age: number };
  type Person = HasName & HasAge;

  interface User {
    readonly id: number;
    name: string;
    email?: string;
  }

  const user: User = { id: 1, name: 'An' };
  user.name = 'Binh';   // OK
  user.id = 2;          // ERROR

  So sánh Flutter:
  - union type = cái Dart không có sẵn
  - readonly ~ final
  - optional property ~ String? email

================================================================================
  4. GENERICS CƠ BẢN
================================================================================

  interface ApiResponse<T> {
    data: T;
    message: string;
  }

  interface Paginated<T> {
    items: T[];
    total: number;
  }

  const productResponse: ApiResponse<Paginated<Product>> = {
    data: {
      items: [],
      total: 0,
    },
    message: 'OK',
  };

  // Angular sẽ gặp rất nhiều:
  // http.get<Product[]>('/api/products')
  // signal<Todo[]>([])
  // input<User>()

  💡 Cách đọc:
  - <T> = cho type vào
  - ApiResponse<Product[]> = response có data là danh sách Product

================================================================================
  5. 3 UTILITY TYPES ĐỦ DÙNG NHẤT
================================================================================

  interface Product {
    id: number;
    name: string;
    price: number;
    description: string;
  }

  type CreateProduct = Omit<Product, 'id'>;
  type UpdateProduct = Partial<Product>;
  type ProductPreview = Pick<Product, 'id' | 'name' | 'price'>;

  Thực tế trong app:
  - Omit: payload tạo mới
  - Partial: payload cập nhật
  - Pick: data nhẹ cho list/card

================================================================================
  6. ĐỌC TYPE ERROR THƯỜNG GẶP
================================================================================

  Error 1:
  Type 'string | undefined' is not assignable to type 'string'

  Nghĩa là:
  - giá trị có thể undefined
  - nhưng nơi đang dùng chỉ chấp nhận string

  Cách xử lý:
  - check if trước
  - đặt default value
  - đổi type nếu domain thực sự cho phép undefined

  Error 2:
  Property 'x' does not exist on type 'Y'

  Nghĩa là:
  - bạn đang đọc sai shape của object
  - cần kiểm tra interface/type của data

  Error 3:
  Type 'A' is not assignable to type 'B'

  Nghĩa là:
  - payload, model hoặc response đang không đúng contract
