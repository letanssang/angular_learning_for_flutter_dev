================================================================================
  STATE MANAGEMENT - NGRX STORE (OPTIONAL / APPENDIX)
================================================================================

THỜI LƯỢNG ƯỚC TÍNH:
  - 10-15 phút

================================================================================
  CHEATSHEET NHANH
================================================================================

  - Action -> Reducer -> Selector -> Effect là chuỗi cơ bản
  - Chỉ học sâu NgRx khi project thực tế dùng hoặc state rất lớn
  - Đừng nhảy vào NgRx trước khi rõ component/service/signals


📌 MỤC TIÊU:
  - Hiểu NgRx giống BLoC/Redux style architecture
  - Đọc được codebase enterprise có action/reducer/effect/selector
  - Không nhầm nó là bắt buộc cho 2 tuần đầu

================================================================================
  1. NGRX = KHI STATE ĐÃ QUÁ LỚN
================================================================================

  ┌──────────────────────────┬──────────────────────────────┐
  │ Flutter BLoC             │ NgRx                         │
  ├──────────────────────────┼──────────────────────────────┤
  │ Event                    │ Action                       │
  │ State                    │ State                        │
  │ Bloc                     │ Reducer                      │
  │ Side effects             │ Effects                      │
  │ BlocSelector             │ Selector                     │
  └──────────────────────────┴──────────────────────────────┘

================================================================================
  2. KHI NÀO MỚI CẦN?
================================================================================

  Chỉ nghiêm túc học NgRx nếu:
  - app lớn, nhiều domain state
  - nhiều dev cùng sửa state layer
  - cần strict unidirectional flow
  - dự án công ty đang dùng sẵn

================================================================================
  3. LỘ TRÌNH NÀY KHUYÊN GÌ?
================================================================================

  Thứ tự ưu tiên:
  1. signal cho local và service-level state đơn giản
  2. Observable cho HTTP/UI streams
  3. NgRx chỉ học khi project thực sự cần

================================================================================
  ✅ TÓM LẠI:
  - NgRx quan trọng trong enterprise Angular
  - Nhưng không phải điểm vào codebase nhanh cho Flutter dev mới chuyển sang
================================================================================
